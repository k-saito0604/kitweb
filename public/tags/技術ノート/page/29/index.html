*技術ノート 2017/10/10* [番号] 技術ノート KGTN 2017090601 [現象] [GGH5.1] HTML5クライアントの通信のプロトコルは？ [説明] HTML5クライアントの通信プロトコルは RXP over HTTP で，より正確に言えば， RXP over WebSocket protocol (ws://) です．暗号化は，この RXP プロトコル上で 56-bit DES または SSL で行われます．詳細については，下記説明をご覧ください． The default protocol for the HTML5 client is RXP over HTTP. This is true, but it's a bit more accurate to say it’s RXP over WebSocket protocol (ws://). By default, GO-Global hosts are configured to use the TCP Transport with 56-bit DES encryption.

SSL is supported though from the HTML5 client but you don't get the secure connection dialog notification on the client (a bug is already logged on the dialog). ファイル KGTN2017090501.pdf [(33.3 KB)] [kitasp 技術センター, 2017/10/10 10:50]
