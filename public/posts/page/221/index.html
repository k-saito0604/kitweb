[GGW3.2.1/GGH4.0] Windows Client でユーザIDとパスワードを指定した時，パスワード不一致だとダイアログ Failed to authenticate user が表示されるだけだが，ログオン画面が表示されないのは仕様か？

in tech

2011-05-24

*技術ノート 2011/05/24* [番号] 技術ノート KGTN 2011052402 [現象] [GGW3.2.1/GGH4.0] Windows Client でユーザIDとパスワードを指定した時，パスワード不一致だとダイアログ “Failed to authenticate user” が表示されるだけだが，ログオン画面が表示されないのは仕様か？ [説明] はい，仕様です．Windows Client でユーザIDとパスワードを指定するケースは，多くの場合管理者がユーザIDを指定したい時です．認証に失敗して 「もし」 ログオン画面を表示したとすると，他のユーザIDでログオン出来ることになりますので，端末とユーザIDの紐付けが崩れてしまいます．このような理由で，ログオン画面を表示しないのが仕様です． ファイル KGTN2011052402.pdf [(49.3 KB)] [kitasp 技術センター, 2011/05/24 09:32]
